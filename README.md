# PirateSwap Skin Watcher

Local Windows UI for watching PirateSwap CS2 listings and sending Discord alerts when a newly appearing watched skin is found.

## Start

Run:

```text
start.bat
```

The launcher will:

- create `.venv` if needed
- install/update Python dependencies
- install/update Playwright Chromium
- open the desktop UI

## UI Flow

1. Paste your Discord webhook URL.
2. Optionally click `Test Discord`.
3. Set the check interval.
4. Add skins to watch:
   - choose `Type`: `Weapon`, `Knife`, or `Gloves`
   - choose the item from the dependent dropdown
   - enter the skin name
   - enable `StatTrak` if applicable
5. Click `Start Watching`.

When the UI opens, `state.json` is reset. The first watcher scan saves the currently visible matches as the baseline and sends no alerts. Later scans compare against that baseline and notify Discord only for newly appearing listings.

## Discord Webhook

In Discord:

1. Create or choose a server/channel.
2. Open channel settings.
3. Go to `Integrations`.
4. Open `Webhooks`.
5. Create a webhook.
6. Copy its URL.
7. Paste it into the app.

Keep the webhook URL private. Anyone with it can post to that Discord channel.

## Runtime Files

The app creates local runtime files:

- `settings.json`: generated by the UI, remembers Discord webhook and interval between app launches
- `state.json`: generated/reset by the UI, used for baseline/new-listing comparison
- `.venv/`: Python virtual environment

These are local machine files and are ignored by git.

## Notes

- PirateSwap URL is fixed in code.
- Browser automation always runs hidden.
- Skins to watch are session-only and must be added each time the UI opens.
- `state.json` always uses the project folder.
- Gloves automatically disable StatTrak.
- Knives and gloves are searched with the Steam/PirateSwap `★` prefix automatically.
